## 一点前言

想做点东西，就想自己写个博客，用的是一个SSG框架Astro做的(这个框架的名字和我的第一辆公路车的名字刚好一样hhh)这里再列举一下SSG，CSR，SSR有什么区别以及适合什么样的场景🤔

以下是new bing的回答：

SSG，CSR，SSR这三种技术都是用于构建和渲染网页的方法，它们各有优点和适用的场景³⁵：

- **SSR（服务器端渲染）Server-Side Generation**：当SEO至关重要并且需要尽快向用户交付内容时，请使用SSR。它非常适合内容驱动的网站和电子商务平台³。

- **CSR（客户端渲染）Client-Side Rendering**：当您需要高度交互和动态的Web应用程序时，请考虑CSR。它非常适合单页应用程序 (SPA) 和严重依赖用户交互的Web应用程序³。

- **SSG（服务器端生成）Server-Side Rendering**：与SSR一样，HTML首先在服务器端生成。SSG和SSR的区别在于HTML是在构建时生成的，而内容是在每次发出请求时从CDN交付的。这用于静态页面，例如博客、帮助站点和电子商务产品列表。SSG具有比SSR更好的性能，因为HTML是在构建时生成的⁵。

我的理解是，SSG的内容是在上服务器之后就是固定了的，而SSR是会根据用户的交互内容在服务器端动态生成HTML以供用户访问，所以适合电商平台这种场景
